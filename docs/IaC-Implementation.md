# 📘 Документация: Восстановление рабочего SSH и создание двух ВМ с уникальными IP через CI/CD

## ✅ Цель

Автоматическое создание двух виртуальных машин Debian 12 в Proxmox с использованием Terraform и Gitea Actions.

**Требования:**
- Уникальные IP-адреса (без конфликтов и дублирования).
- Доступ по SSH с ключом, заданным через Gitea Secrets.
- Полная автоматизация — после `git push` машины создаются и готовы к работе.

---

## 🧩 Исходное состояние (до исправлений)

- Шаблон Proxmox: ID 199 (`debian12-cloud-template-ntrokhin`), cloud‑init с пользователем `ansible`.
- CI/CD: Gitea Actions запускает `terraform apply` при изменениях в `terraform/**`.
- **Проблема:** При создании двух ВМ обе получали одинаковый IP (`192.168.77.160`) и SSH‑доступ работал нестабильно (ключи не совпадали или отсутствовали).

---

## ⚙️ Финальная рабочая конфигурация

### 1. Terraform (`main.tf`)

- Генерация случайного VMID в диапазоне 165–195.
- Клонирование шаблона 199.
- Назначение статического IP на основе VMID: `192.168.77.<VMID>/24`, шлюз `192.168.77.100`.
- Блок `user_account` создаёт пользователя `ansible` и передаёт публичный SSH‑ключ из переменной `var.ssh_public_key` (читается из секрета Gitea).
- Размер диска явно указан `30G` (совпадает с шаблоном).

### 2. Terraform (`variables.tf`)

- Добавлена переменная `ssh_public_key` (sensitive), принимающая значение из `TF_VAR_ssh_public_key`.

### 3. Gitea Actions Workflow (`.gitea/workflows/terraform.yml`)

- Во всех шагах `Terraform Init`, `Plan`, `Apply` экспортируется переменная окружения:
  ```bash
  export TF_VAR_ssh_public_key="${{ secrets.SSH_PUBLIC_KEY }}"
