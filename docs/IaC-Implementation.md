# Автоматизация развёртывания ВМ в Proxmox через CI/CD

## Цель

Настроить автоматическое создание двух виртуальных машин Debian 12 в Proxmox при каждом `git push` с уникальными IP-адресами и готовым SSH-доступом.

## Проблема

При использовании Terraform и Gitea Actions возникали конфликты:
- Обе ВМ получали одинаковый IP (`192.168.77.160`)
- SSH-доступ не работал из-за несовпадения ключей

## Решение

- Перешёл со статического DHCP на динамическое назначение IP, привязанное к VMID: `192.168.77.<VMID>/24`
- Настроил передачу публичного SSH-ключа через Gitea Secrets в переменную `TF_VAR_ssh_public_key`
- Добавил явное указание размера диска (`30G`) для соответствия шаблону
- Внёс правило обязательного форматирования кода `terraform fmt -recursive` перед коммитом

## Результат

- ВМ создаются полностью автоматически после `git push`
- IP-адреса уникальны, конфликты исключены
- SSH-доступ работает сразу, без ручного вмешательства
- Код и секреты разделены, инфраструктура готова к повторному использованию

## Используемые инструменты

Terraform, Proxmox VE, Gitea Actions (CI/CD), Gitea Secrets, Debian 12 Cloud-Init.
